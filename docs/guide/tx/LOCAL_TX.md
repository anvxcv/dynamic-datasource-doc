# 本地多数据源事务

## 背景

多数据源事务方案一直是一个难题，通常的解决方案有二种。

1. 利用atomiks手动构建多数据源事务，适合数据源较少，配置的参数也不太多的项目。难点就是手动配置量大，需要耗费一定时间。
2. 用seata类似的分布式事务解决方案。难点就是需要搭建如seata-server的统一管理中心。

每一种方案都有其适用场景，本项目作者常常收到的问题就是。
1. 为什么事务下切换数据源失败？ 
2. 有没有不依赖第三方的方案？
